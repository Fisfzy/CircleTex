# 更新记录

## 0.7.3

- 普通文字选区随请求携带受限长度的左右可见上下文，在源码存在重复短语时先通过上下文进行后端唯一消歧；
- 上下文不足时使用 SyncTeX 正向定位候选页码与空间位置，并与实际 PDF 选区比较；仍不唯一时继续拒绝，不接受前端指定源码偏移；
- 保留源码哈希、队列版本、重叠修改、Unicode 字素边界及公式、命令参数、表格和注释环境保护；
- 在修改要求区域下方增加自动编译进度条，展示校验、备份、写入、预检、每遍 XeLaTeX、收敛判断、产物发布和 PDF 刷新阶段；
- XeLaTeX 阶段使用动画状态，不伪造单遍内部百分比；成功显示绿色 100% 后自动收起，失败保留红色状态和原错误信息；
- 增加重复短语上下文消歧、SyncTeX 正向输出、编译阶段进度、失败状态及窄窗口布局回归测试。

## 0.7.2

- 红色新增标签增加插入点竖线、细引线和三角箭头，缩放、滚动与虚拟重渲染后仍保持锚定；
- PDF 刷新改为并发扫描页面元数据、保留旧当前页快照并优先渲染当前页，减少编译完成后的空白等待；
- 缓存已成功解析的 XeLaTeX、SyncTeX、PowerShell、Codex 和 Snow 命令路径，相关 CLI 配置变化时自动失效；
- 编译使用上一轮辅助文件预热隔离构建目录；仅在辅助文件完全稳定且日志没有重跑警告时省略默认第二遍，否则保持完整两遍；
- CircleTeX 日志增加预检、命令定位、每遍 XeLaTeX、产物发布、PDF 读取、元数据扫描和当前页渲染耗时；
- AI 提示词将选中源码从前后只读上下文中剔除，避免重复传输，同时保持 Codex 与 Snow 的会话隔离策略不变。

## 0.7.1

- 光标插入在完整 PDF 上下文因 `\\cite`、行内公式或 `\\%` 等渲染差异无法直接匹配时，使用自适应双侧短锚点定位普通正文边界；
- 双侧锚点必须形成唯一且相邻的源码位置；重复短语、命令边界、命令参数、数学和注释环境仍保守拒绝，删除与替换不放宽；
- 调整 AI 分析区布局，修改要求输入框位于左侧，当前助手分析按钮位于右侧，窄窗口下自动换行。

## 0.7.0

- 默认启用直接编辑模式：待编译标记只存在于 PDF 覆盖层，写入 `main.tex` 时直接完成增删改，最终 PDF 不保留红色或删除线；
- 增加 PDF 铅笔工具，支持拖选后键盘替换、`Backspace`/`Delete` 删除、单击文字间隙插入，以及光标前删和后删；
- 增加中文输入法组合输入、`Ctrl+Enter` 提交、`Esc` 取消、`Ctrl+Z` 撤销和 `Ctrl+Y`/`Ctrl+Shift+Z` 重做；
- 增加可选“查看改动”，直接模式不再强制打开差异页或二次确认，仍保留备份、基线校验、后台编译和失败回滚；
- 保留可选的修订痕迹模式；检测到 `0.6.0` 遗留修订时，要求先接受全部或拒绝全部，禁止与直接编辑混合写入；
- 光标定位改用唯一文字上下文和 Unicode 字素簇边界，完整处理组合重音、Hangul Jamo、变体选择符和 ZWJ 序列；
- 扩展公式、表格和逐字环境保护，拒绝在 `equation`、`align`、`tabular`、`verbatim` 等不安全上下文中直接编辑；
- 左侧设置增加 PDF 编辑模式选择，并对待编译队列锁定写入模式，配置切换不会改变已有队列语义；
- 编译入口拒绝未保存的 `main.tex`，产物覆盖后、事务提交前会再次校验磁盘源码和编辑器状态；校验失败时恢复旧 PDF 与 SyncTeX。

## 0.6.0

- 增加 PDF 普通文字手动修订：前插、后插、替换和删除；
- 新增内容以红色锚定标签显示，删除和替换范围使用淡红底与红色中划线，缩放、滚动和虚拟重渲染后保持稳定；
- 增加后端权威修订队列、版本校验、撤销、清空、重叠拒绝和过期基线保护；
- 手动编译时先打开完整 TeX 差异并二次确认，成功后才清空队列；失败时安全恢复源码并保留待提交修订；
- 编译后的 PDF 保留红色修订标记，并支持接受全部或拒绝全部；
- 使用自包含、可换行的中划线宏，不依赖额外安装 `ulem`；
- 修复取消写入后的按钮恢复、选区切换竞态和编译结束时的源码再校验，并保证窄窗口下工具栏仍可访问；
- 增加 18 项字符映射与修订宏单元测试、中文 XeLaTeX 宏烟测和手动修订 Webview 烟测。

## 0.5.0

- 编译改为后台运行，不再自动展开输出面板或弹出成功通知，失败时提供显式日志入口；
- 自动修订预览在旁侧打开但保留 PDF 焦点，点击“查看差异”时才主动聚焦；
- 左侧设置增加 AI 助手选择和编译日志入口；
- 新增 Snow CLI ACP 适配器，在独立临时目录中禁用工具、MCP、文件系统和终端能力；
- Codex CLI 与 Snow CLI 共用结构化结果校验、差异预览和人工确认写入链路。

## 0.4.1

- 修复磁盘混合换行被 VS Code 统一后错误触发“建议生成后发生变化”的问题；
- 写入前分别核对磁盘原文与编辑器文本，并使用磁盘原文创建备份；
- 目标范围外发生变化时重新定位并刷新差异，要求用户再次确认后才写入；
- 目标内容变化、两侧复杂变化或重复源码片段存在歧义时继续拒绝应用。

## 0.4.0

- 增加区域框选工具，可在单个 PDF 页面上绘制矩形，并提取区域内命中的文字；
- 使用独立覆盖层呈现稳定高亮，缩放、滚动及页面重新渲染后仍可恢复选区；
- 每个视觉行生成受限锚点并执行多点 SyncTeX 源码顺序检查，区域范围始终需要人工确认；
- 支持按 `Esc` 或使用清除选区按钮取消区域框选；
- 区域框选限定为单页矩形，只处理 PDF 文字层内容，不提取扫描图片中的文字。

## 0.3.0
