# Word 版式重建规范

## 目标边界

以 `main.tex` 保存内容和结构语义，以论文文档类保存学校格式规则，以 `main.pdf` 作为只读验收参照。正式 DOCX 应满足格式规范一致和结构完整；由于 Word 与 TeX 的断行、浮动体和分页算法不同，不要求逐像素或逐页完全一致。

不得把 Pandoc 默认 DOCX 当作最终版式。Pandoc 只负责将 LaTeX 正文转换为 Word 可识别的段落、标题、列表、图表和占位符。

## 确定性流水线

1. 解析 `main.tex`，提取元数据、公式、标题层级、图表数量和前置结构。
2. 根据 `assets/hrbeu-midterm-word-layout.json` 动态生成内部 `reference.docx`；该文件是 CircleTeX 的构建资产，不是用户提供的 Word 底稿。
3. 使用 `--reference-doc` 生成基础 DOCX，禁止直接交付 Pandoc 默认样式。
4. 装配全部 MathType OLE 对象，保持零 OMML、零图片或文本公式降级。
5. 在 Word 中重建封面、目录和分节，设置目录罗马页码、正文阿拉伯页码、首页无页眉页脚，并恢复页眉横线、标题、正文和题注样式。
6. 将 LaTeX 强制分页标记转换为 Word 分页符，随后更新目录、域和分页。
7. 重开 DOCX，执行公式完整性门禁和版式结构门禁。

## 规则归属

- 页面、字体、段落、标题、题注、页眉和封面参数统一写入 `assets/hrbeu-midterm-word-layout.json`，脚本不得另存一套相互冲突的常量。
- `prepare_tex_for_word.py` 只提取结构和生成稳定标记，不负责猜测 Word 版式。
- `word_layout.ps1` 负责生成参考模板、重建固定结构和审计 Word。
- `export_mathtype_word.ps1` 只编排阶段和汇总报告。
- Agent 只能解释失败或为未知宏提出映射建议，不得逐段人工重排正式 DOCX。

## 强制门禁

- 公式：MathType 对象数量与 LaTeX 公式位置数量一致；OMML、残留占位符和公式降级数均为零。
- 页面：所有节均为 A4，四边距和页眉页脚距离符合版式配置。
- 结构：声明了封面时必须存在独立无页眉页脚封面节；声明了目录时必须存在 Word 自动目录；正文必须是独立节。
- 页码：目录节采用大写罗马数字并从 I 开始；正文节采用阿拉伯数字并从 1 开始。
- 标题：Word 中一级至四级标题数量不得少于 LaTeX 中对应标题数量。
- 字段：装配 MathType 和版式后必须更新目录与字段，再保存并重开验收。

图片数量、表格数量和最终页数写入报告用于比较。复杂浮动体会因排版引擎差异改变分页，因此当前作为显式警告，不单独导致公式合格的文件失败；数量缺失必须在报告中指出，后续不得宣称为高度视觉复刻稿。
